# TODO Sprint 1 (Oppsett og grunnmur)

Maal for sprinten:
- Programmet starter lokalt.
- Database-tilkobling fungerer.
- Teamet bruker samme Git/GitHub-rutine.

Prioritet: Hoy = maa gjores i sprint 1, Medium = bor gjores i sprint 1.

## Oppgaver

- [ ] Hoy: Avklare roller i gruppa (DB, GUI, API, dokumentasjon) og hvem som eier hva i sprint 1.
- [ ] Hoy: Lage en felles arbeidsavtale for GitHub (alltid pull for start, commit ofte, push daglig).
- [x] Hoy: Opprette grunnstruktur i prosjektet (for eksempel mapper for gui, db, api, docs).
- [ ] Hoy: Verifisere at alle i gruppa kan clone repoet og pushe en test-commit.
- [ ] Hoy: Sette opp Python-miljo og avhengigheter som gruppa skal bruke.
- [ ] Hoy: Lage og teste database-tilkobling fra Python.
- [ ] Hoy: Lage en enkel DB-test (for eksempel SELECT 1 eller lesing av 1 tabell) med tydelig feilmelding ved feil.
- [ ] Hoy: Lage en enkel GUI-startside (vindu med tittel + minst en knapp/meny).
- [ ] Hoy: Koble GUI til en enkel handling (for eksempel "Test DB") som viser om DB svarer.
- [ ] Hoy: Lage grunnleggende input-validering (tom streng, ugyldige tegn) i minst ett skjema.
- [ ] Hoy: Sjekke at SQL-kall bruker parametre (ikke string-konkatenering) i all ny kode.
- [ ] Medium: Lage enkel logg/feilhaandtering slik at appen ikke krasjer ved vanlige feil.
- [ ] Medium: Opprette en krav-sjekkliste for videre sprint (hva som er gjort/ikke gjort).
- [ ] Medium: Starte rapportutkast med korte notater: valg av GUI-teknologi og hvorfor.
- [ ] Medium: Oppdatere gruppeloggen med hvem som gjorde hva denne sprinten (basert pa commits).

## Definition of Done for Sprint 1

- [ ] Appen starter uten krasj.
- [ ] DB-tilkobling fungerer pa minst to maskiner i gruppa.
- [ ] Minst ett GUI-element er koblet til en fungerende handling.
- [ ] Git-rutinen er testet av alle (pull -> add -> commit -> push).
- [ ] Alle oppgaver markert som Hoy er ferdige eller flyttet med begrunnelse.

## Forslag til fordeling i oppstartsmote (30 min)

- [ ] 10 min: Fordel roller og oppgaver.
- [ ] 10 min: Sett opp miljo og test DB-tilgang.
- [ ] 10 min: Lag forste GUI-skjelett og push forste commit.
